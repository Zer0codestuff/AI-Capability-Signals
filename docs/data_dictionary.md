# Data Dictionary

## Core Tables

- `models_master.csv`: union of normalized Epoch AI model metadata and OpenRouter public catalog rows.
- `models_epoch_normalized.csv`: historical model metadata from Epoch AI with parsed dates, scale fields and access classification.
- `models_openrouter_normalized.csv`: public API catalog rows with prices normalized to USD per 1M tokens.
- `pricing_openrouter.csv`: compact pricing/context table.
- `benchmarks_lmarena_latest.csv`: selected latest LMArena leaderboard parquet snapshots.
- `benchmarks_livebench.csv`: extracted LiveBench benchmark rows when the public schema exposes a score.
- `benchmarks_swebench_verified.csv`: SWE-bench Verified public submissions, resolved count and percent score.
- `oracle_release_events.csv`: release events with calendar, moon and planetary features.
- `oracle_pattern_tests.csv`: random-baseline tests for the release-pattern appendix.
- `sources.csv`: source registry and capture metadata.

## Entity Fields

- `vendor`: canonical organization/vendor where it can be inferred.
- `model_family`: product/model family such as GPT, Claude, Gemini, Gemma, Qwen, Llama or Other.
- `product_line`: narrower product line when available. It is currently equal to the detected family for most rows.
- `family`: compatibility alias for `model_family`; new code should prefer `model_family`.

## Rich Dataset Tables

The wide dataset package lives in `data/dataset/` and is generated by:

```bash
uv run python -m frontier_ai.dataset_factory
```

It includes Hugging Face model metadata, model repository files, full LMArena snapshots, Open LLM Leaderboard metrics, OpenAlex paper metadata, GitHub ecosystem metadata, SWE-bench instance outcomes, and a unified model index.

## Access Classes

- `open_weight`: source explicitly says open weights.
- `likely_open_weight`: model family strongly implies downloadable/open-weight release, but the row did not explicitly say open weights.
- `closed_or_api`: proprietary or API-first release.
- `unknown`: insufficient public signal.

The project avoids using `open_source` as a blanket label because many open-weight models do not release training code, data or permissive licenses.
